다음은 지금까지 정리한 **‘테일로그’ 최종 와이어프레임 + 실현 가능성·허점 보완 + 다음 단계(DB 설계 제안)까지 반영한 버전**이다.[1][2]

## 0. 서비스 핵심 컨셉 요약

- 목표: **데이터 기반 ABC 행동 수정 + AI 코칭**을 웹(PWA)으로 제공하는 반려견 행동 교정 SaaS.[1]
- 스택: Next.js + Tailwind + shadcn/ui, PWA, Kakao Sync/알림톡, RAG 기반 AI 코칭.[1]
- BM: Freemium (웹 푸시 중심) + Pro (알림톡·고급 리포트·AI 코칭 확장).[1]

  - `/dog/profile` 반려견 프로필/환경 정보 수정

## 2. 랜딩 `/`

**헤더**

- 좌측: 로고(테일로그)  
- 우측: [기능] [가격] [FAQ] [로그인]

**Hero**

- 텍스트:
  - H1: “머루의 행동, 데이터로 코칭받으세요”
  - 서브: “3분 설문으로 맞춤 행동 리포트를 무료로 받아보세요. 앱 설치 없이, 웹에서 바로.”[1]
- 버튼:
  - Primary: “머루 행동 무료 체크하기” → `/checkup`
  - Secondary: “이미 계정이 있어요” → `/login`
- 비주얼:
  - Lottie 또는 루프 영상(강아지+보호자 훈련 장면)

**문제·솔루션 섹션**

- 카드: “매번 같은 영상”, “효과가 있는지 모르겠음”, “얼마나 나아졌는지 불분명”
- 솔루션: “맞춤형 코칭”, “데이터 리포트”, “알림톡(구독자 전용) 리마인더”[1]

**사용 흐름 섹션**

- Step 1: 설문 3분
- Step 2: 맞춤 리포트
- Step 3: 매일 기록 + AI 코칭
- Step 4: Pro 구독 시 알림톡·전문 리포트

**가격 섹션**

- Free vs Pro 카드 (Pro에 “알림톡 포함” 배지).[1]

## 3. 게스트 온보딩 설문 `/checkup`

### 공통 요소

- 상단: 진행률 바 + 텍스트  
  - “현재 20% 완료! 머루의 성격 분석까지 2분 남았어요.”  
- 하단: [이전] [다음] 버튼  
- **중간 저장 UX(허점 보완)**:
  - 단계 3~4 진입 시:  
    - “지금까지 입력한 내용을 저장할까요?  
       카카오로 연결하면 나중에 이어서 볼 수 있어요.”  
    - 버튼: [카카오로 저장하기] → 소프트 로그인 (선택 사항).

***

### 단계 1. 기본 정보

- 이름, 성별, 견종, 나이(생년월일/대략), 몸무게, 중성화.[1]

### 단계 2. 환경

- 가족 구성, 혼자 있는 시간, 주거 형태, 쉬는 공간.[1]

### 단계 3. 건강·영양

- 건강 상태, 질병/수술/알레르기, 식단, 식사 루틴, 식사 중 행동(자원 지키기 등).[1]

### 단계 4. 문제 행동

- 우선순위 1~3개 선택:
  - 초인종/문 앞 짖음, 요구성 짖음, 자원 지키기, 하우스 회피, 배변 문제, 분리불안 등.[1]
- 각 문제별 상세 질문(빈도, 지속시간, 촉발 자극, 현재 대처, 메모).[1]

### 단계 5. 산책·놀이

- 산책 빈도·시간, 현재 산책 어려움(예방접종/불안 등), 줄 종류, 놀이 종류·패턴, 장난감 관리.[1]

### 단계 6. 성격·기질

- 한 줄 성격 요약, 낯선 환경/사람/동물/소음/접촉에 대한 반응, 집중력, 보호자 애착 정도.[1]

## 4. 결과 페이지 `/result`

**요약 헤더**

- “머루는 이런 아이예요”  
- 태그: “예민한 문지기”, “자원 지키기 경향”, “실내 생활 중심” 등.

**문제 행동 요약**

- 문제별 카드:
  - 심각도(별 1~5), 빈도, 상황, 요약 설명, “AI 해석” 한 줄.

**오늘 시작할 3가지 행동**

- “오늘은 여기부터” 체크리스트:
  - 예: “문 소리 들리면 3m 거리 두기 연습 3회” 등.

**행동 지도(Behavior Map) 맛보기**

- 간단 인포그래픽:
  - “머루는 평일 저녁 시간과 현관 근처에서 특히 민감합니다.”  
  - Time x Location 간단 시각 레이아웃.[1]

**회원가입 유도**

- “이 리포트를 저장하고, 머루의 변화를 매주 보고 싶다면?”  
- Free/Pro 기능 요약 +  
- 버튼:  
  - [계정 만들고 계속하기] → `/login`  
  - 작은 링크: “그냥 결과만 보고 나갈게요”

## 5. 로그인/회원가입 `/login`

- 카카오 버튼: “카카오로 시작하기”
- 이메일 폼(옵션)
- 온보딩 중간 저장과 연결:
  - 온보딩 중 카카오로 저장했다면, 완료 후 자동 로그인 & 데이터 연동.

## 6. 메인 대시보드 `/dashboard`

**헤더**

- 좌측 로고  
- 가운데 반려견 선택(머루)  
- 우측: 알림 아이콘, 설정 아이콘.

**Hero**

- 배경 이미지/영상(머루 또는 샘플)  
- 텍스트:
  - “오늘도 머루와 1% 나아지는 하루를”  
  - “연속 7일 기록 중이에요.”
- 버튼: [오늘 퀵 기록하기] → `/log`

**요약 카드**

- 오늘 기록 수, 이번 주 달성률, 스트릭, 다음 배지까지.[1]

**오늘의 미션 + 체크리스트(연속성 강화)**

- “오늘 할 것” 카드:
  - 예: “문 소리-거리 두기 연습”, “하우스 긍정 연습” 등 3개.
  - 각각 체크박스.
- 체크하면:
  - 대시보드 상단에 “오늘 미션 2/3 완료”  
  - /analytics에서 실제 짖음 강도·빈도와 연결해 변화 분석.[1]

**히트맵 + 트렌드**

- 월간 히트맵(기록 여부)  
- 주간 짖음 빈도 그래프.

**AI 피드백 요약**

- “어제 머루의 행동 분석”  
- 3~4줄 요약 + [자세히 보기] → `/coach`.

## 7. 기록 페이지 `/log` – 퀵 + ABC

### 상단

- “지금 상황을 빠르게 남기거나, 자세하게 기록할 수 있어요.”

### 섹션 1. 퀵 버튼(피로도 보완)

- 큰 버튼: “지금 짖었어요!”  
  - 클릭 시:
    - 시간 자동 저장
    - 위치(GPS, 선택)  
    - [나중에 상세 입력하기] 안내  
    - DB에는 A/B/C 미완성 상태로 `quick_event`로 저장.[1]

- 추가 퀵 버튼:
  - “지금 불안해하는 것 같아요”
  - “지금 누군가에게 입질했어요”

### 섹션 2. 상세 ABC 기록

- Step A: 선행 사건
- Step B: 행동(유형/강도/지속시간)
- Step C: 보호자 반응
- 완료 시 Lottie + “오늘 n번째 기록입니다.”

***

## 8. 분석 페이지 `/analytics` – 행동 지도 포함

### 탭1: 달력(히트맵)

- 월간 기록 여부 히트맵 + 요약 지표.

### 탭2: 주간 분석

- 짖음/입질/배변 등 문제 행동 그래프.
- 인사이트 텍스트:
  - “월·수·금 저녁에 짖음이 집중됩니다” 등.[1]

### 탭3: 기록 목록

- 필터 + 카드 목록.

### 탭4: **행동 지도(Behavior Map)**

- 시간대(축) × 상황/장소(축) Heatmap:
  - 예:  
    - X축: 시간대(0~24시)  
    - Y축: 상황/장소(현관, 거실, 산책, 혼자 있을 때 등)[1]
- 요약:
  - “머루는 평일 저녁 집 안 현관 근처에서 가장 예민합니다.”[1]

## 9. AI 코칭 페이지 `/coach`

**일간 피드백**

- “어제의 머루 리포트”  
- 요약 + 오늘 한 가지 추천.

**주간 리포트**

- Pro: PDF 다운로드, 수의사/훈련사 공유용.[1]
- 버튼: “QR 코드로 공유하기”
  - `/report/:id` 읽기 전용 링크 → 전문가에게 보여주기(B2B 확장).[1]

**AI 상담 채팅**

- 사용자 질문 입력  
- 백엔드는:
  - ABC 로그 + 환경 + 기질 데이터 → RAG로 불러와 개인화된 답변 생성.[1]

## 12. 실현 가능성·개발 우선순위

- 기술적 실현성: 높음 (Next.js PWA + Kakao + Supabase + RAG).[1]
- **MVP에서 집중할 것:**
  1) 온보딩(축약 버전 + 마이크로 카피 + 중간 저장)  
  2) ABC 기록(퀵 버튼 포함)  
  3) 간단한 AI 피드백(/coach의 일간 요약 + 간단 조언).[1]
- Lottie/영상/고화질 이미지는 **핵심 전환 포인트(온보딩 완료, 기록 완료, 리포트 로딩 등)에만 전략적으로 배치**.[3][4]
